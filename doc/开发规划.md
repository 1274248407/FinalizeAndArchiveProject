我有一个已完成漫画翻译的归档目录 `D:\documents\translation\02_Archive_Completed`。所有在该目录下的文件夹都严格遵循 `YYYY-MM-DD_漫画名称` 的命名规范。其中开头的日期是“立项时间”，而非翻译完成时间。

由于我通常采用**“剪切/移动”（Move）**的方式将翻译好的文件夹放进归档目录，在 Windows 文件系统中，这会导致文件夹的 `CreationTime`（创建时间）仍保留为很久以前的立项时间，而**只有 `LastWriteTime`（最后修改时间）能真实反映其翻译完成/最后导出的时间**。

为了保持归档目录的整洁，我需要一个 pwsh7 脚本，自动删除**归档（即翻译完成）超过 30 天**的文件夹，并在执行后发送系统通知。

具体逻辑与技术要求如下：

1. **时间判定基准（核心防错）**：
   - **绝对不能**使用文件名中的 `YYYY-MM-DD` 日期。
   - 考虑到“剪切/移动”的文件系统特性，请使用**文件夹本身的 `LastWriteTime`** 作为归档时间的判定基准（如果能安全地检测文件夹内所有文件的“最大/最新 LastWriteTime”作为辅助双重保险则更好）。
   - 计算公式：(当前系统时间 - 文件夹最后修改时间) > 30 天，则判定为需要清理。
2. **强制递归删除**：
   - 判定删除后，必须使用 `Remove-Item -Recurse -Force` 强制删除该文件夹及其内部的所有子文件和子文件夹。
   - 必须使用 `-LiteralPath` 以完美支持漫画文件夹中常见的各种特殊字符（如中括号 `[]`、圆括号 `()`、波浪号 `~` 等），避免路径解析报错。
3. **安全与控制机制（配置化）**：
   - 脚本顶部提供可配变量：`$TargetDir = "D:\documents\translation\02_Archive_Completed"` 和 `$DaysToKeep = 30`。
   - 默认开启“安全模拟（WhatIf）”模式（可以通过变量控制如 `$ForceDelete = $false`）。在模拟模式下，控制台仅打印删除预估，不执行真实删除。
   - 必须使用 `Try-Catch` 块包裹删除执行部分，确保遇到文件占用等异常时能记录日志并继续执行，不中途崩溃。
4. **执行后通知（Win10 原生 Toast 弹窗）**：
   - 脚本执行完成后，**无论是否有文件夹被删除，均需发送通知**。
   - 必须使用 **Windows 10 原生的 Toast 弹窗通知机制**（通过调用 UWP XML 接口 `[Windows.UI.Notifications.ToastNotificationManager]` 编写纯 PowerShell 实现，无需安装任何第三方模块如 BurntToast），确保通知在弹出后能**保留在 Windows 右下角的“通知中心（Action Center）”**，方便我后续查看。
   - 通知内容模板：
     - 若有文件被清理（或在模拟模式下发现过期文件夹）：
       - “漫画归档清理完成：已成功清理 [X] 个超过 30 天的翻译文件夹。”
       - 模拟模式：“[模拟运行] 发现 [X] 个可清理的过期文件夹。”
     - 若没有文件符合清理条件：
       - “漫画归档状态：未发现超过 30 天的过期文件夹，无需清理。”
