## 依赖评估阈值

达到以下任意条件时，重新评估是否引入第三方模块：

### 引入 [Logging](https://www.powershellgallery.com/packages/Logging) 模块的时机

| 阈值 | 说明 |
|------|------|
| 函数数量 10+ | 日志追踪变复杂，手动格式化不够用 |
| 需要文件日志 | 排查问题需要持久化日志 |
| 多模块协作 | 需要统一日志格式和级别 |
| 调试耗时增加 | 开始频繁用 `Write-Host` 打 debug 信息 |

### 引入 [BurntToast](https://www.powershellgallery.com/packages/BurntToast) 模块的时机

| 阈值 | 说明 |
|------|------|
| 需要图片/按钮/交互 | 自己写的纯文本通知不够用 |
| 通知场景 3+ | 多种通知模板需要维护，BurntToast 的 API 更简洁 |
| 需要跨用户通知 | BurntToast 支持 `-Credential` 参数指定目标用户 |
